# Project Objective

This file gives agents a stable mission and a shared technical bar.
Replace every placeholder before using this template for a real project.

## Mission

Build and evolve `<project-name>` so it delivers `<primary-user-outcome>` reliably,
maintainably, and with clear verification.

## Expected Result

The system should be able to:

- accept the main project inputs
- apply the core domain workflow
- interact with required external systems through explicit boundaries
- produce the expected user-facing output
- expose enough traces, tests, or artifacts to verify behavior

## Technical Objective

The project should remain understandable and safe to change:

- entrypoints compose dependencies
- application/use-case code orchestrates behavior
- domain code owns business rules and stable contracts
- infrastructure code implements external adapters
- tests and docs stay aligned with implementation

## Quality Bar

Every change should optimize for:

- clear responsibilities
- low coupling
- deterministic behavior where possible
- explicit error handling and fallbacks
- proportionate tests and documentation
